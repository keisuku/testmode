更新テンプレート（コピペ用）

（例: world.md を最新の価格データで更新して）

### 2. 対象ファイル
（例: knowledge/world.md）

### 3. 変更内容（要点3つ）
1.
2.
3.

### 4. 出力形式
（ diff / full file / 該当セクションのみ ）

### 5. コミットメッセージ
（例: "Update world.md: add new stage Shanghai"）

コンテンツテンプレート — 新規Markdownファイルのひな形

最短結論

新しいナレッジファイルを作成する際はこのテンプレートに従う。俳句圧縮スタイルを厳守。

テンプレート

---
title: "タイトル"
tags: ["tag1", "tag2"]
status: "draft"
updated: "YYYY-MM-DD"
author: "claude"
related: ["related-file.md"]
---

# タイトル — サブタイトル（1行で内容を要約）

## 最短結論

（1〜2文で結論を述べる。これだけ読めば要点が分かるように。）

---

## 大見出し1

### 中見出し

- 要点1
- 要点2
- 要点3

## 大見出し2

### 中見出し

- 要点

フォーマットルール

俳句圧縮スタイル

  1. 最短結論（1行）を冒頭に置く
  2. 大見出し → 中見出し → 要点（箇条書き） の階層
  3. 冗長な説明は避け、要点を箇条書きで圧縮
  4. 表は情報の比較・一覧に積極的に使う

Frontmatter 必須フィールド

フィールド 説明 例
title ファイルのタイトル “世界観設定”
tags タグ配列 [“world”, “setting”]
status draft / wip / stable “wip”
updated 最終更新日 “2026-03-11”
author 著者 “claude”
related 関連ファイル [“saki-info.md”]

固有名詞ルール

以下は絶対に変更しない:

  • 高市総理
  • 片山財務大臣
  • 小野田オタク大臣
  • 鉄腕アトムのプルート

CoinBattleSaki — 仮想通貨Botバトルとキャラクターが融合したプロジェクト

最短結論

CoinBattleSaki は、リアルタイムの仮想通貨先物データを使った Botバトル系プロジェクト。AI生成キャラ・画像・演出とトレード戦略をゲーム化し、世界観・ルール・自動化を統合する。

プロジェクト概要

本質

  • リアルタイムの仮想通貨先物データを用いた Botバトル 系プロジェクト
  • AI生成要素（キャラクター、画像、演出）とトレード戦略をゲーム化する設計
  • 世界観・キャラ・ルール・AI生成を統合したプロジェクト
  • ドキュメントは常に「読みやすく・俳句圧縮スタイル」で整える

データソース

  • 仮想通貨先物の リアルタイムティック/板/約定データ を前提とする
  • HFT（高頻度取引）系の短時間足重視

時間軸（重要）

時間足 優先度
1分 最高
3分 高
5分 高
15分 中
30分 中
1時間 標準

短時間軸を多めにする方針。

キャラクター/設定

デザイン方針

  • 狐（女性イメージ） を中心としたキャラクター体系
  • 各キャラに短いメタデータ（性別/時間軸/イメージ/ロール）を付与

主要キャラクター

雷狐（Raikō）

  • 性別: 女性
  • イメージ: 細身・長手・切れ長の顔
  • ロール: 狐系メインキャラクター

  • ロール: HFT EMPEROR
  • イメージ: 高頻度取引の帝王
  • 備考: HFT戦略のシンボル的存在

世界観運用

  • ステージは 多様（世界各地） で展開
  • 北斗の拳 や ストリートファイター2 のように場が多様であるべき
  • パチンコ的演出要素も検討対象

演出方針

  • ゲーム的な演出と実データを融合
  • AI生成画像・動画をバトル演出に活用
  • パチンコ的な「激熱」「確変」のような演出レイヤーを検討

ツール/環境

開発基盤

  • GitHub: ソースコード管理・ファイル差分管理
  • 画像生成AI: キャラクター・ステージ画像の自動生成
  • 動画生成AI: バトル演出動画の自動生成

設計方針

  • Notion を無理に使うより Markdown/HTML Knowledge Base を Claude が編集する形が望ましい
  • 速度・安定性重視 の設計思想

制約/注意点

  • 俳句圧縮スタイル: 短い結論 → 見出し → 要点（箇条書き）

トレーディングシステム — HFT重視の仮想通貨先物Botバトル

最短結論

仮想通貨先物のリアルタイムデータを使い、短時間軸（1分〜1時間）で Bot 同士がバトルする。HFT戦略を中心に据える。

データ仕様

入力データ

  • ティックデータ: リアルタイム価格更新
  • 板情報: オーダーブックの深度
  • 約定データ: 実行された取引の履歴

時間軸設定

時間足 用途 優先度
1分 スキャルピング 最高
3分 超短期トレンド 高
5分 短期モメンタム 高
15分 中期トレンド 中

1時間 全体方向性 標準

Bot 戦略タイプ

スキャルパー型

  • 時間軸: 1分〜3分
  • 特徴: 高頻度・小利確・損切り即座
  • 代表キャラ: 雷狐（Raikō）

HFT型

  • 時間軸: ティック〜1分
  • 特徴: マシン精度の高速執行
  • 代表キャラ: 霧島 一郎（Kirishima Ichiro）

モメンタム型

  • 時間軸: 5分〜15分
  • 特徴: トレンドに乗る順張り
  • 代表キャラ: （追加予定）

スイング型

  • 時間軸: 30分〜1時間
  • 特徴: 大きな波を捉える
  • 代表キャラ: （追加予定）

バトルルール（仮）

  • 同一時間帯に複数Botが同一銘柄でトレード
  • 一定期間後の損益で勝敗を判定
  • 連勝ボーナス・逆転演出あり

デプロイ手順 — 静的サイトの公開方法

最短結論

./build.sh で site/ を生成し、GitHub Pages / Netlify / Cloudflare Pages にデプロイする。

ビルド

# Node.js でビルド
./build.sh

# または Python でビルド
./build.sh --python

成果物: site/index.html

環境変数設定

注意: APIキー等のセンシティブ情報はリポジトリに含めないこと。

必要に応じて以下の環境変数を設定:

# .env（.gitignore に含めること）
SITE_TITLE="CoinBattleSaki Knowledge Base"
SITE_URL="https://your-domain.com"

デプロイ先オプション

GitHub Pages

  1. リポジトリの Settings → Pages
  2. Source: Deploy from a branch
  3. Branch: main / フォルダ: /site
  4. または GitHub Actions で自動ビルド・デプロイ
# .github/workflows/deploy.yml の例
name: Deploy
on:
  push:
    branches: [main]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm install
      - run: ./build.sh
      - uses: actions/upload-pages-artifact@v3
        with:
          path: site/

Netlify

  1. netlify.toml を作成:
[build]
  command = "./build.sh"
  publish = "site/"
  1. Netlify にリポジトリを接続
  2. 自動デプロイが有効化される

Cloudflare Pages

  1. Cloudflare Dashboard → Pages → Create a project
  2. Git リポジトリを接続
  3. Build command: ./build.sh
  4. Output directory: site/

セキュリティチェックリスト

  • [ ] .env ファイルが .gitignore に含まれている
  • [ ] APIキーがソースコードに含まれていない
  • [ ] 公開ファイルにセンシティブ情報がない
  • [ ] HTTPS が有効化されている

最短結論

Markdown Knowledge Base + 静的サイト生成 + Git 管理。Claude Code で自動更新し、AI生成と仮想通貨データを統合する。

データフロー

ナレッジ更新

  1. テンプレコピペ → Claude に依頼
  2. Claude が knowledge/*.md を編集
  3. git commit で差分記録
  4. ./build.sh で site/ を再生成

バトルデータ（将来）

  1. 仮想通貨先物API → リアルタイムデータ取得
  2. Bot エンジンがトレード実行
  3. バトル結果を knowledge/ に記録
  4. AI生成で演出画像/動画を生成

ファイル命名規則

  • knowledge/: 小文字スネークケース（例: trading_system.md）
  • docs/: 小文字スネークケース
  • tools/: build_ プレフィックス + 言語名
  • web/: 標準的なWeb命名規則
