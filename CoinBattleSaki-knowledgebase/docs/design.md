# 設計ドキュメント — CoinBattleSaki 全体アーキテクチャ

## 最短結論

Markdown Knowledge Base + 静的サイト生成 + Git 管理。Claude Code で自動更新し、AI生成と仮想通貨データを統合する。

---

## 設計原則

1. **速度優先**: Notion より Markdown/HTML が速い
2. **安定性**: Git ベースで差分管理、ロールバック可能
3. **自動化**: Claude Code が編集→コミット→ビルドまで一貫
4. **拡張性**: ファイル追加だけで知識が増える

## データフロー

### ナレッジ更新

1. テンプレコピペ → Claude に依頼
2. Claude が `knowledge/*.md` を編集
3. `git commit` で差分記録
4. `./build.sh` で `site/` を再生成

### バトルデータ（将来）

1. 仮想通貨先物API → リアルタイムデータ取得
2. Bot エンジンがトレード実行
3. バトル結果を `knowledge/` に記録
4. AI生成で演出画像/動画を生成

## ファイル命名規則

- `knowledge/`: 小文字スネークケース（例: `trading_system.md`）
- `docs/`: 小文字スネークケース
- `tools/`: `build_` プレフィックス + 言語名
- `web/`: 標準的なWeb命名規則
