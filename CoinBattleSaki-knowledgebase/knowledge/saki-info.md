# CoinBattleSaki — 仮想通貨Botバトルとキャラクターが融合したプロジェクト

## 最短結論

CoinBattleSaki は、リアルタイムの仮想通貨先物データを使った **Botバトル系プロジェクト**。AI生成キャラ・画像・演出とトレード戦略をゲーム化し、世界観・ルール・自動化を統合する。

---

## プロジェクト概要

### 本質

- リアルタイムの仮想通貨先物データを用いた **Botバトル** 系プロジェクト
- AI生成要素（キャラクター、画像、演出）とトレード戦略をゲーム化する設計
- 世界観・キャラ・ルール・AI生成を統合したプロジェクト
- ドキュメントは常に「読みやすく・俳句圧縮スタイル」で整える

### データソース

- 仮想通貨先物の **リアルタイムティック/板/約定データ** を前提とする
- HFT（高頻度取引）系の短時間足重視

### 時間軸（重要）

| 時間足 | 優先度 |
|--------|--------|
| 1分    | 最高   |
| 3分    | 高     |
| 5分    | 高     |
| 15分   | 中     |
| 30分   | 中     |
| 1時間  | 標準   |

短時間軸を多めにする方針。

## キャラクター/設定

### デザイン方針

- **狐（女性イメージ）** を中心としたキャラクター体系
- 各キャラに短いメタデータ（性別/時間軸/イメージ/ロール）を付与

### 主要キャラクター

#### 雷狐（Raikō）
- **性別**: 女性
- **イメージ**: 細身・長手・切れ長の顔
- **ロール**: 狐系メインキャラクター
- **備考**: 狐は女性イメージで統一

#### 霧島 一郎（Kirishima Ichiro）
- **ロール**: HFT EMPEROR
- **イメージ**: 高頻度取引の帝王
- **備考**: HFT戦略のシンボル的存在

## 世界観運用

### ステージ設計

- ステージは **多様（世界各地）** で展開
- **北斗の拳** や **ストリートファイター2** のように場が多様であるべき
- パチンコ的演出要素も検討対象

### 演出方針

- ゲーム的な演出と実データを融合
- AI生成画像・動画をバトル演出に活用
- パチンコ的な「激熱」「確変」のような演出レイヤーを検討

---

## ツール/環境

### 開発基盤

- **GitHub**: ソースコード管理・ファイル差分管理
- **画像生成AI**: キャラクター・ステージ画像の自動生成
- **動画生成AI**: バトル演出動画の自動生成
- **Claude Code**: 自動化ワークフローの中核
- **MCP ターミナル接続**: 実験段階（完全ではない）

### 設計方針

- Notion を無理に使うより **Markdown/HTML Knowledge Base** を Claude が編集する形が望ましい
- **速度・安定性重視** の設計思想
- Git によるファイル差分管理を活用

---

## 制約/注意点

### 固有名詞の保全（絶対厳守）

以下の固有名詞は **絶対に変換・改名しない**:

- **高市総理**
- **片山財務大臣**
- **小野田オタク大臣**
- **鉄腕アトムのプルート**

### 文章スタイル

- 出力は「ブログに転記しても読み物として通用する」整った文章スタイル
- ユーザー（keisuku様）は「**まず最短結論 → 詳細**」の構成を好む
- 俳句圧縮スタイル: 短い結論 → 見出し → 要点（箇条書き）

---

## 参考機能要求

- ファイル差分管理（Git）を活用
- 各キャラに短いメタデータ（性別/時間軸/イメージ/ロール）を付与
- `prompts/claude_update.md` に Claude に渡す更新テンプレを用意（1回コピペで更新可能）
- センシティブ情報（APIキー等）はファイルに書かない
- 環境変数設定手順は `docs/deploy.md` に記載
